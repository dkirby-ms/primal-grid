## Task

Investigate and fix visibility filtering bug — players could see entire map despite fog of war implementation.

## Root Cause

Missing `@view()` decorator on `tiles` ArraySchema field in `GameState`. Colyseus 0.17 requires `@view()` on the collection field to activate StateView per-client filtering. Without it:
- `encoder.context.hasFilters` stays `false`
- `SchemaSerializer` ignores `client.view` entirely
- Full state is broadcast to all clients

## Resolution

Added `@view()` decorator to the `tiles` field in `GameState`:

```typescript
@type([TileState])
@view()
tiles = new ArraySchema<TileState>();
```

This reverses the earlier decision of "NO @view() field decorators" (see decisions.md for full context). The earlier decision was based on a misunderstanding — `@view()` on the field doesn't filter the field itself, it enables per-element filtering within the collection via `view.add(item)` / `view.remove(item)`.

## Outcome

✅ Root cause identified and fixed  
✅ All 372 tests pass unchanged  
✅ Build clean, zero lint errors  
✅ Server-side StateView filtering now active  

## Key Correction

**Design decision reversal:** `@view()` IS needed on the parent collection field to enable StateView filtering. Earlier assumption that this would filter the entire field was incorrect. The decorator activates the filtering pipeline; the per-element filtering still happens via `view.add()/remove()` calls.
