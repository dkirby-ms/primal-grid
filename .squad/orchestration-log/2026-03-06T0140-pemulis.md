## Design Decisions Documented

1. **Hysteresis exhaustion:** Stamina ≤ 0 → exhausted state persists until stamina ≥ threshold. Prevents rapid idle↔move flickering.
2. **Movement-only deduction:** Stamina only deducted on actual tile moves, not blocked attempts.
3. **Builder stamina in PAWN namespace:** Builders use PAWN.STAMINA_* constants, unified via `getStaminaConfig()`.
4. **Regen on idle:** Stamina regens during all non-movement states, decoupling movement cost from food cost.
5. **Exhausted state skips FSM:** Exhausted creatures don't flee/hunt — only regen. Creates tactical predator windows.
