## Findings & Learnings

### Manhattan vs. Euclidean
Initial implementation used Euclidean distance. Tests failed until corrected to Manhattan (`|dx| + |dy| <= radius`). This matches the `addCircleFill` implementation used in actual visibility computation.

### Edge-Vision Union Edge Case
Side-edge tiles (e.g., hqX+2, hqY) are only Manhattan distance 2 from HQ center. With edge radius 3, they reach exactly HQ radius 5 — not beyond. **Corner edge tiles** (Manhattan distance 4 from HQ) are needed to prove edge vision extends visibility beyond HQ radius.

### StateView Object Parent Requirement
`StateView.add()` throws if object has no parent ChangeTree. Tiles in `state.tiles` ArraySchema already have parents (assigned during map generation), so calls are safe. Test setup must call `room.generateMap()` before attempting `view.add()`.

### onLeave Cleanup
`cleanupPlayerView()` must be called in `onLeave()` callback to prevent memory leak. Test verified with 10 join/leave cycles: `playerViews.size === 0` after all complete.
