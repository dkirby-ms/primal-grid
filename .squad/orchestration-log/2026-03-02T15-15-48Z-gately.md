- Space considerations: carousel at 180px width, ~70px height, fits within 600px panel with room to spare

## Design Rationale

**Why always visible:** Improves shape discoverability. New players see all available shapes immediately — no hidden menu. Removal of B-key also removes hidden state from UX vocabulary.

**Why below creatures:** Natural reading order (top = status/inventory, bottom = gameplay units). Carousel height scales gracefully as player levels (2 shapes L1 → 7 shapes L5 wraps into 2 rows).

**Why gold highlight, not build banner:** The selected shape in carousel IS the visual indicator. Gold border + subtle background (existing `.selected` CSS) is sufficient. Removes `#build-indicator` banner clutter.

## Interaction Model: Selection Not Mode

| Action | Result |
|--------|--------|
| Click shape in carousel | Selects it. Ghost preview on grid. Cursor → `cell`. |
| Click same shape again | Deselects. Ghost clears. Cursor → `crosshair`. |
| Number keys 1–9 | Select shape by index (shortcut to armed state). |
| Q/E | Cycle prev/next shape (no-op if nothing selected). |
| R | Rotate (no-op if nothing selected). |
| Click grid (shape armed) | Place shape. Stay armed for rapid placement. |
| Click grid (no shape armed) | No-op. |
| Escape | Deselect. |
| Right-click | Deselect (prevent context menu). |

## Risk Mitigation

**Accidental placement risk:** Ghost preview shows exactly where shape will land. Players see consequences before clicking. Natural cost (2 wood/cell) mitigates mis-clicks.

**Discoverability:** Always-visible carousel + number key shortcuts + hint bar reduce learning curve vs. hidden build mode behind B key.
