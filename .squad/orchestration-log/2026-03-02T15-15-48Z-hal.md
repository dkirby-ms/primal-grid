## Design Rationale

**Why "stay armed":** Builder games (Factorio, RimWorld, Satisfactory) all keep selection live after placement for rapid click-click-click. Manual reselection would be infuriating.

**Why no mode:** Carousel visibility + selection highlight is sufficient UX. Removes cognitive load (no mode to remember), improves discoverability (shapes always visible = new players see them immediately), matches industry standard.

**Why zero server risk:** Build mode is 100% client-side. Server validates `PLACE_SHAPE { shapeId, x, y, rotation }` independently. Any client interaction redesign producing the same message is backward-compatible.

## Architecture Insight

Build mode is a client-only state machine. Server has zero concept of it. This design simply replaces the state variable (`buildMode: boolean` → `selectedShapeIndex: number | null`) and updates handlers accordingly.
