## Key Design Decisions

1. **No `@view()` decorators** — element-level `view.add/remove` sufficient for two-tier visibility (visible vs. not visible)
2. **No owned-tile cache** — deferred to Phase 2 performance optimization (not critical for current 64×64 maps)
3. **Lazy playerViews initialization** — guards for `Object.create()` test pattern (test suite manually initializes `playerViews = new Map()`)
4. **StateView.add() safe after map generation** — tiles in `state.tiles` ArraySchema already have parent ChangeTree

## Learnings for Other Agents

- StateView API: `view.add(obj)` / `view.remove(obj)` / `view.has(obj)` from `@colyseus/schema` v4.0.16
- Test pattern `Object.create(GameRoom.prototype)` skips constructor — any new class fields need lazy `??` or `if (!this.x)` guards
- Manhattan distance for circle fill: `|dx| + |dy| <= radius` produces diamond-shaped vision (not euclidean circles)
- Reconnection handling: Not yet specified in design; test suite assumes single connection per player
